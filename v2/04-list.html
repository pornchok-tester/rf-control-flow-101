บทที่ 4

List ใน Robot Framework

✅ RF 4.0+

🎯 อ่านจบแล้วทำได้

  • เข้าใจว่า List คืออะไรและเก็บอะไรได้บ้าง
  • สร้าง List และดูจำนวน item ด้วย Get Length
  • วนผ่านทุก item ด้วย FOR IN
  • เชื่อมต่อความรู้นี้กับ Get WebElements ในบทถัดไปได้

ทำไมต้องรู้เรื่อง List?

ในบทถัดไปจะเจอ keyword ชื่อ Get WebElements — keyword นี้คืนค่ากลับมาเป็น List ของ element บนหน้าเว็บ ถ้าไม่รู้ว่า List คืออะไร จะงงตั้งแต่บรรทัดแรก

บทนี้สอนแค่ที่จำเป็นสำหรับบทถัดไป

List คืออะไร?

List คือ กล่องที่เก็บของหลายชิ้นเรียงกัน เหมือน checklist ทดสอบที่มีหลาย step — แต่ละ step มีเลขที่ของตัวเอง

จุดสำคัญ: เลขที่ของ item ใน List (เรียกว่า index) เริ่มจาก 0 เสมอ ไม่ใช่ 1

@{fruits} = [apple, banana, cherry]
index 0 = apple  |  index 1 = banana  |  index 2 = cherry

สร้าง List

ใช้ keyword Create List แล้วใส่ค่าที่ต้องการเรียงกัน:

*** Test Cases ***
ตัวอย่าง List
    @{fruits}=    Create List    apple    banana    cherry
    Log    ${fruits}    # [apple, banana, cherry]
    Log    ${fruits}[0]    # apple
    Log    ${fruits}[1]    # banana
    Log    ${fruits}[2]    # cherry
Output
[apple, banana, cherry]
apple
banana
cherry
ตอนสร้าง list ใช้ @{} — ตอนดึงค่าออกมาใช้ ${}

@{fruits}= Create List ...  ← สร้าง
Log ${fruits}[0]  ← ดึง item ที่ index 0
FOR ${x} IN @{fruits}  ← วน

ดูจำนวน item — Get Length

@{browsers}=    Create List    Chrome    Firefox    Edge
${count}=    Get Length    ${browsers}
Log    มี ${count} browser    # มี 3 browser

FOR IN — วนผ่านทุก item

นี่คือสิ่งที่เรียนไปในบทที่ 2 แล้ว แค่ตอนนี้รู้ว่าสิ่งที่ FOR วนผ่านนั้นคือ List นั่นเอง:

FOR IN — วนผ่าน list ทีละ item

List ว่าง — ต้องระวัง

ถ้า List ไม่มี item เลย FOR จะไม่ทำงานเลย ไม่มี error — เงียบๆ และผ่านไปเลย:

@{empty}=    Create List    # ไม่มี item
FOR    ${x}    IN    @{empty}
    Log    บรรทัดนี้ไม่มีทางรัน
END
Log    มาถึงนี่เลย โดยไม่ผ่าน loop
ถ้าต้องการให้ test fail เมื่อ List ว่าง ให้เช็คก่อน:

Should Not Be Empty ${my_list} ไม่พบข้อมูลใดเลย

เชื่อมกับบทถัดไป

Get WebElements ใน SeleniumLibrary ทำงานแบบเดียวกันทุกอย่าง — แค่แทนที่จะสร้าง list เอง มันดึง element จากหน้าเว็บมาให้เป็น list แล้วเราก็วนผ่านด้วย FOR เหมือนกันเลย:

# แทนที่จะสร้าง list เอง...
@{fruits}=    Create List    apple    banana    cherry

# Get WebElements ดึง list จากหน้าเว็บมาให้เลย
@{buttons}=    Get WebElements    css=.submit-button

นั่นคือทั้งหมดที่ต้องรู้ก่อนไปบทถัดไป

🧠 เช็คความเข้าใจ

ข้อ 1. List มี 5 item — item สุดท้ายอยู่ที่ index อะไร?

ดูเฉลย
index 4 — เพราะ index เริ่มจาก 0 ดังนั้น 5 item = index 0, 1, 2, 3, 4

ข้อ 2. ถ้า List ว่างแล้ว FOR วนผ่าน จะเกิดอะไรขึ้น?

ดูเฉลย
ไม่มี error — FOR แค่ไม่ทำงานเลยเงียบๆ ถ้าต้องการให้ fail เมื่อว่าง ต้องใช้ Should Not Be Empty เองก่อน loop

ข้อ 3. ต่างกันยังไงระหว่าง @{list} กับ ${list}?

ดูเฉลย
@{list} ใช้ตอน FOR เพื่อแตก list ออกเป็น item ทีละตัว | ${list} ใช้ตอนอ้างอิง list ทั้งก้อน เช่น Get Length ${list} หรือ ${list}[0]
