ส่วนที่ 1: IF / ELSE

📝 Recall
ข้อ 1 — ลำดับ Condition

ถ้าเขียน IF/ELSE IF โดยเรียง condition จากน้อยไปมาก เช่น:

IF    ${temp} > 10
    Log    อุ่น
ELSE IF    ${temp} > 30
    Log    ร้อน
END

ถ้า ${temp} = 35 จะ log อะไร? มีปัญหาอะไรกับโค้ดนี้ไหม? อธิบายด้วยคำตัวเอง

ดูเฉลย

log "อุ่น" — เพราะ 35 > 10 เป็น True ตั้งแต่ condition แรก RF จะข้าม ELSE IF ไป

ปัญหา: ไม่มีทางได้ "ร้อน" เลย ควรเรียงจาก "เข้มงวดที่สุด" ก่อน:

IF    ${temp} > 30
    Log    ร้อน
ELSE IF    ${temp} > 10
    Log    อุ่น
END
💻 Application
ข้อ 2 — Classify File Size

เขียน keyword ชื่อ Classify File Size ที่รับ argument ${size_mb} แล้ว log ว่า:

  • น้อยกว่า 1 MB → "Small"
  • 1–100 MB → "Medium"
  • มากกว่า 100 MB → "Large"

พร้อมเขียน test case ที่ call keyword นี้ด้วย 3 ค่าที่ครอบคลุมทุก branch

ดูเฉลย 
*** Keywords ***
Classify File Size
    [Arguments]    ${size_mb}
    IF    ${size_mb} < 1
        Log    Small
    ELSE IF    ${size_mb} <= 100
        Log    Medium
    ELSE
        Log    Large
    END

*** Test Cases ***
Test File Size Classification
    Classify File Size    0.5
    Classify File Size    50
    Classify File Size    200
🔬 Synthesis
ข้อ 3 — หา Bug ใน Check Login Status

ดูโค้ดต่อไปนี้แล้วหา bug — มีปัญหากี่จุด? แต่ละจุดเกิดอะไรขึ้น? เขียนโค้ดที่แก้แล้วด้วย

Check Login Status
    [Arguments]    ${username}    ${role}
    IF    ${role} == admin
        Log    Welcome Admin
    ELSE IF    ${role} == user
        Log    Welcome User
    ELSE
        Log    Unknown role
    END
ดูเฉลย

Bug 2 จุด:

  • ${role} == admin → ขาด quote รอบ string ทั้งสองฝั่ง
  • ${role} == user → เช่นเดียวกัน
Check Login Status
    [Arguments]    ${username}    ${role}
    IF    '${role}' == 'admin'
        Log    Welcome Admin
    ELSE IF    '${role}' == 'user'
        Log    Welcome User
    ELSE
        Log    Unknown role
    END

ส่วนที่ 2: FOR Loop

📝 Recall
ข้อ 4 — FOR IN vs FOR IN RANGE

อธิบายว่า FOR IN กับ FOR IN RANGE ต่างกันยังไง และควรใช้อันไหนในสถานการณ์ต่อไปนี้ (ไม่ต้องมีโค้ด):

  • วนผ่านรายชื่อ browser ทุกตัวที่มีใน list
  • ลอง submit form สูงสุด 5 ครั้ง
ดูเฉลย

FOR IN — วนผ่านทุก item ใน list ที่มีอยู่ ไม่รู้ล่วงหน้าว่ามีกี่ตัวก็ได้

FOR IN RANGE — วนตามจำนวนรอบที่กำหนด รู้ล่วงหน้าว่าจะวนกี่ครั้ง

วนผ่าน browser list → FOR IN เพราะมี list อยู่แล้ว

ลอง submit 5 ครั้ง → FOR IN RANGE 5 เพราะรู้จำนวนรอบล่วงหน้า

💻 Application
ข้อ 5 — นับ Pass / Fail

เขียน keyword ชื่อ Log Test Summary ที่รับ @{results} (list ของ "Pass" หรือ "Fail") แล้ว:

  • Log แต่ละผลลัพธ์ทีละบรรทัด
  • Log สรุปท้าย: สรุป: Pass X | Fail Y

ทดสอบกับ Pass Fail Pass Pass Fail (expect: Pass 3 | Fail 2)

ดูเฉลย 
*** Keywords ***
Log Test Summary
    [Arguments]    @{results}
    ${pass_count}=    Set Variable    0
    ${fail_count}=    Set Variable    0
    FOR    ${r}    IN    @{results}
        Log    ${r}
        IF    '${r}' == 'Pass'
            ${pass_count}=    Evaluate    ${pass_count} + 1
        ELSE
            ${fail_count}=    Evaluate    ${fail_count} + 1
        END
    END
    Log    สรุป: Pass ${pass_count} | Fail ${fail_count}

*** Test Cases ***
Test Log Summary
    Log Test Summary    Pass    Fail    Pass    Pass    Fail
Output
Pass
Fail
Pass
Pass
Fail
สรุป: Pass 3 | Fail 2
🔬 Synthesis
ข้อ 6 — หา Bug ใน Check All Approved

ดูโค้ดนี้แล้วหา bug 2 จุด — แต่ละจุดทำให้เกิดอะไร? และเขียนโค้ดที่แก้แล้วด้วย

Check All Items Approved
    [Arguments]    @{items}
    FOR    ${item}    IN    @{items}
        IF    '${item}' == 'Rejected'    BREAK
    END
    Log    ทุก item ผ่านแล้ว
ดูเฉลย

Bug 1: BREAK หยุด loop ได้ แต่ Log ทุก item ผ่านแล้ว อยู่นอก loop — ถ้าเจอ "Rejected" ก็ยัง Log ว่าผ่านอยู่ดี

Bug 2: ถ้า list ว่าง loop ไม่ทำงานเลย แต่ยัง Log "ทุก item ผ่านแล้ว" → false positive

Check All Items Approved
    [Arguments]    @{items}
    Should Not Be Empty    ${items}    ไม่มี item ให้ตรวจ
    ${rejected}=    Set Variable    ${False}
    FOR    ${item}    IN    @{items}
        IF    '${item}' == 'Rejected'
            ${rejected}=    Set Variable    ${True}
            BREAK
        END
    END
    Should Be True    not ${rejected}    พบ item ที่ถูก Rejected

ส่วนที่ 3: WHILE + BREAK

📝 Recall
ข้อ 7 — WHILE limit

อธิบายว่า limit parameter ของ WHILE มีไว้ทำอะไร ถ้าไม่ใส่จะเกิดอะไรขึ้นเมื่อ condition ไม่เคยเป็น False?

ดูเฉลย

limit กำหนดจำนวนรอบสูงสุดที่ WHILE จะวน ถ้าไม่ใส่ default คือ 10,000 รอบ

ถ้า condition ไม่เคยเป็น False และไม่มี BREAK → loop จะวนจนครบ limit แล้ว FAIL ทันที (ไม่ใช่ค้างตลอดไป)

💻 Application
ข้อ 8 — Find First Even

เขียน keyword ชื่อ Find First Even ที่รับ @{numbers} แล้ว:

  • วน loop ผ่านทุกตัวเลข
  • ถ้าเจอเลขคู่ → log ว่า "Found even: [number]" แล้วหยุด
  • ถ้าไม่มีเลขคู่เลย → log ว่า "No even number found"

ทดสอบกับ ${1} ${3} ${7} ${4} ${9} (expect: หยุดที่ 4)

ดูเฉลย 
*** Keywords ***
Find First Even
    [Arguments]    @{numbers}
    ${found}=    Set Variable    False
    FOR    ${num}    IN    @{numbers}
        ${is_even}=    Evaluate    ${num} % 2 == 0
        IF    ${is_even}
            Log    Found even: ${num}
            ${found}=    Set Variable    True
            BREAK
        END
    END
    IF    '${found}' == 'False'
        Log    No even number found
    END

*** Test Cases ***
Test Find First Even
    Find First Even    ${1}    ${3}    ${7}    ${4}    ${9}
Output
Found even: 4
🔬 Synthesis
ข้อ 9 — วิเคราะห์ Retry Until Success

ดูโค้ดต่อไปนี้แล้วตอบ 3 คำถาม:

Retry Until Success
    [Arguments]    ${max_attempts}
    ${attempt}=    Set Variable    1
    WHILE    ${attempt} <= ${max_attempts}
        ${result}=    Run Keyword And Return Status    Flaky Keyword
        IF    ${result}    BREAK
        ${attempt}=    Evaluate    ${attempt} + 1
    END
    Should Be True    ${result}    Keyword failed after ${max_attempts} attempts
  1. ถ้า Flaky Keyword สำเร็จในรอบที่ 2 จาก 5 — ${attempt} จะเป็นค่าอะไรหลัง BREAK?
  2. ถ้า Flaky Keyword ล้มเหลวทุกรอบ — Should Be True จะได้รับค่าอะไร? test จะ pass หรือ fail?
  3. มีจุดไหนที่อาจเป็น bug ไหม? (hint: ลองคิดถึงกรณีที่ max_attempts = 0)
ดูเฉลย

ข้อ 1: ${attempt} = 2 — BREAK หยุด loop แต่ ${attempt} ยังมีค่าของ iteration ที่ BREAK เกิดขึ้น

ข้อ 2: ${result} = False → Should Be True False → test FAIL พร้อมข้อความ "Keyword failed after N attempts" — ซึ่งถูกต้อง

ข้อ 3: Bug — ถ้า max_attempts = 0 → WHILE 1 <= 0 = False ตั้งแต่แรก loop ไม่ทำงานเลย แต่ ${result} ยังไม่ถูก assign → Should Be True ${result} จะ error เพราะ ${result} ไม่มีค่า

Fix: เพิ่ม ${result}= Set Variable ${False} ก่อน WHILE

Retry Until Success
    [Arguments]    ${max_attempts}
    ${attempt}=    Set Variable    1
    ${result}=     Set Variable    ${False}    # ← initialize ก่อน WHILE
    WHILE    ${attempt} <= ${max_attempts}
        ${result}=    Run Keyword And Return Status    Flaky Keyword
        IF    ${result}    BREAK
        ${attempt}=    Evaluate    ${attempt} + 1
    END
    Should Be True    ${result}    Keyword failed after ${max_attempts} attempts
