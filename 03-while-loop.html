บทที่ 3

WHILE Loop, BREAK และ CONTINUE

⚠️ WHILE / BREAK / CONTINUE ต้องใช้ RF 5.0+

🎯 อ่านจบแล้วทำได้

  • เขียน WHILE loop ที่วนซ้ำตามเงื่อนไข
  • ตั้ง limit เพื่อป้องกัน infinite loop
  • ใช้ BREAK ออกจาก loop กลางคัน
  • ใช้ CONTINUE ข้าม iteration ปัจจุบัน

ทำไม WHILE ถึงต้องมีเพิ่มมาจาก FOR?

FOR เหมาะตอนที่รู้ว่าต้องวนกี่รอบ แต่ถ้าต้องรอ element โหลด, poll API จนกว่า status จะเปลี่ยน, หรือ retry จนกว่าจะผ่าน — จำนวนรอบบอกไม่ได้ล่วงหน้า นั่นคืองานของ WHILE

🚪 Analogy: พนักงานเฝ้าด่าน
  • ก่อนให้รถผ่าน → เช็คเงื่อนไข (WHILE condition)
  • ถ้าเงื่อนไขจริง → ทำงาน (รัน keyword ข้างใน)
  • วนกลับมาเช็คใหม่ทุกรอบ
  • ถ้าเงื่อนไขเป็น False → หยุด ออกจาก loop

BREAK เหมือนโทรศัพท์ฉุกเฉินที่บังคับให้หยุดทันที | CONTINUE เหมือนโบกให้รถผ่านไปก่อนโดยไม่ตรวจ แล้วรอคันถัดไป

⚠️ ถ้าเชื่อ analogy นี้ 100% จะเข้าใจผิดว่า: "WHILE รัน loop body ก่อนแล้วค่อยเช็ค" — จริงๆ RF เช็ค condition ก่อนเสมอ ถ้า False ตั้งแต่แรก loop body จะไม่ทำงานเลยแม้แต่รอบเดียว

1. WHILE พื้นฐาน

*** Test Cases ***
WHILE Basic
    ${counter}=    Set Variable    0
    WHILE    ${counter} < 3
        Log    Counter: ${counter}
        ${counter}=    Evaluate    ${counter} + 1
    END
Output
Counter: 0
Counter: 1
Counter: 2
❌ ลืมอัปเดต condition variable — infinite loop!
${x}=    Set Variable    0
WHILE    ${x} < 10
    Log    วนตลอดกาล!
    # ลืม update ${x} → วนจนถึง limit แล้ว FAIL
END
✅ อัปเดตตัวแปรใน loop body เสมอ
    ${x}=    Evaluate    ${x} + 1
source: RF User Guide

2. Limit — ป้องกัน Infinite Loop

WHILE มี iteration limit default = 10,000 รอบ เมื่อถึงจะ FAIL ทันที:

รูปแบบความหมาย
limit=100สูงสุด 100 รอบ
limit=10 secondsสูงสุด 10 วินาที
limit=NONEไม่มี limit (ระวัง! ต้องมี BREAK)
*** Test Cases ***
WHILE With Limit
    ${counter}=    Set Variable    0
    WHILE    True    limit=5
        Log    run ${counter}
        ${counter}=    Evaluate    ${counter} + 1
        IF    ${counter} >= 3    BREAK
    END

3. BREAK — ออกจาก Loop ทันที

*** Test Cases ***
BREAK Demo
    FOR    ${i}    IN RANGE    10
        IF    ${i} == 4
            Log    เจอ 4 แล้ว หยุดเลย
            BREAK
        END
        Log    ${i}
    END
Output
0
1
2
3
เจอ 4 แล้ว หยุดเลย

BREAK ทำงานได้ทั้งใน FOR และ WHILE

4. CONTINUE — ข้าม Iteration ปัจจุบัน

*** Test Cases ***
CONTINUE Demo
    FOR    ${i}    IN RANGE    6
        IF    ${i} % 2 == 0    CONTINUE
        Log    เลขคี่: ${i}
    END
Output
เลขคี่: 1
เลขคี่: 3
เลขคี่: 5

CONTINUE ข้าม keyword ที่เหลือใน iteration นั้น แล้วไปเริ่ม iteration ถัดไปทันที

5. ตัวอย่าง Production: รอ Element โหลด

*** Keywords ***
Wait Until Page Ready
    [Documentation]    Poll จนกว่า element จะพร้อม (สูงสุด 30 วินาที)
    ${found}=    Set Variable    False
    WHILE    '${found}' == 'False'    limit=30 seconds
        ${status}=    Run Keyword And Return Status
        ...    Element Should Be Visible    id=main-content
        IF    ${status}
            ${found}=    Set Variable    True
        ELSE
            Sleep    1s
        END
    END

Common Mistakes

❌ คิดว่า limit=NONE ปลอดภัย
WHILE    True    limit=NONE    # ถ้าเงื่อนไขไม่เปลี่ยน test ค้างไปเรื่อยๆ
✅ ใช้ limit=NONE เฉพาะเมื่อมั่นใจว่า BREAK จะทำงาน
source: RF User Guide
❌ ใช้ BREAK/CONTINUE นอก loop
BREAK    # ← error! ต้องอยู่ภายใน FOR หรือ WHILE
✅ BREAK และ CONTINUE ใช้ได้เฉพาะภายใน loop body
source: RF User Guide

🧠 ทดสอบความเข้าใจ — ตอบก่อนดูเฉลย

ข้อ 1. ถ้า WHILE condition เป็น False ตั้งแต่เริ่มต้น loop body จะทำงานกี่รอบ?

ดูเฉลย
0 รอบ — RF เช็ค condition ก่อนเสมอ ถ้า False ตั้งแต่แรก ข้ามทั้ง loop ไปเลย

ข้อ 2. ต่างกันอย่างไรระหว่าง BREAK กับ CONTINUE?

ดูเฉลย

BREAK หยุด loop ทั้งหมด ออกจาก loop ทันที

CONTINUE หยุดเฉพาะ iteration ปัจจุบัน แล้ววนไปเริ่ม iteration ถัดไป

ข้อ 3. เมื่อไหร่ควรใช้ WHILE แทน FOR? ยกตัวอย่างสถานการณ์จริง 1 อย่าง

ดูเฉลย
ใช้ WHILE เมื่อไม่รู้จำนวนรอบล่วงหน้า เช่น: รอ background job เสร็จ, poll API จนได้ status = "completed", รอ element หายจากหน้าเว็บ
