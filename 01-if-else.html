บทที่ 1

IF / ELSE IF / ELSE และ Inline IF

✅ RF 4.0+ (Inline IF: RF 5.0+)

🎯 อ่านจบแล้วทำได้

  • เขียน IF block ที่ทำงานตามเงื่อนไข
  • ต่อ ELSE IF และ ELSE สำหรับหลายกรณี
  • ใช้ Inline IF เมื่อต้องการโค้ดสั้นกว่า
  • หลีกเลี่ยง mistake ที่พบบ่อยที่สุดใน IF

ทำไมต้องมี IF?

ลองนึกภาพ test ที่ต้องรันได้ทั้ง production และ staging แต่ค่าที่ต้อง check ต่างกัน ถ้าไม่มี IF ก็ต้องก็อป test ไปเขียนแยก 2 ไฟล์ แก้อะไรก็ต้องแก้ 2 ที่ นั่นคือปัญหา

🚦 Analogy: ไฟจราจร

IF/ELSE เหมือนไฟจราจร — ไฟเขียว (IF true) รถวิ่ง, ไฟเหลือง (ELSE IF) ชะลอ, ไฟแดง (ELSE) หยุด

โค้ดอ่านเงื่อนไขจากบนลงล่าง เจอ condition แรกที่จริง → ทำ แล้วข้ามอันที่เหลือทั้งหมด

⚠️ ถ้าเชื่อ analogy นี้ 100% จะเข้าใจผิดว่า: "RF หยุดทำงานหลัง IF" — จริงๆ RF ยังรัน keyword ที่อยู่นอก IF block ต่อไปหลัง END เสมอ

1. IF Block พื้นฐาน

IF    ${score} >= 90
    Log    ผ่านดีมาก
END

กฎ: ต้องมี END ปิดทุกครั้ง ถ้าไม่มี RF จะ error ทันที

2. IF + ELSE IF + ELSE

*** Test Cases ***
Grade Check
    ${score}=    Set Variable    75

    IF    ${score} >= 90
        Log    A
    ELSE IF    ${score} >= 80
        Log    B
    ELSE IF    ${score} >= 70
        Log    C
    ELSE
        Log    F
    END
Output
C

RF อ่านจากบนลงล่าง เจอ condition แรกที่ True ก็รันแล้วออกเลย ไม่ไปเช็ค ELSE IF ที่เหลือ

💡 Tip: เรียง condition จากเข้มงวดที่สุด

ถ้าเรียง condition จากน้อยไปมาก เช่น >= 70 ก่อน >= 90 คะแนน 95 จะเข้า branch C ทันที ไม่มีทางได้ A เลย

3. Condition กับตัวแปรสตริง

เมื่อ compare string ต้องใส่ single quote รอบ ${variable}:

*** Test Cases ***
Environment Check
    ${env}=    Set Variable    production

    IF    '${env}' == 'production'
        Log    กำลังทดสอบบน Production!
    ELSE
        Log    Environment: ${env}
    END
❌ ผิด — ขาด quote รอบ string
IF    ${env} == 'production'
✅ ถูก — ใส่ quote ทั้งสองฝั่ง
IF    '${env}' == 'production'
เหตุผล: ${env} ถูก resolve เป็น string ก่อน evaluate — การใส่ quote ทำให้มั่นใจว่า compare string กับ string เสมอ

4. Inline IF (RF 5.0+)

ใช้เมื่อต้องการรัน keyword เดียว ตามเงื่อนไข — ไม่ต้องมี END:

*** Test Cases ***
Inline IF Demo
    ${x}=    Set Variable    10

    # รูปแบบพื้นฐาน — keyword เดียว
    IF    ${x} > 5    Log    x มากกว่า 5

    # มี ELSE ได้
    IF    ${x} > 5    Log    big    ELSE    Log    small

    # รับค่า return ได้
    ${label}=    IF    ${x} > 5    Set Variable    big    ELSE    Set Variable    small
    Log    ${label}    # → big
เมื่อไหร่ใช้ตัวเลือก
Keyword เดียว, condition เดียว, logic ไม่ซับซ้อนInline IF
มี ELSE IF, หรือหลาย keyword ใน branchBlock IF + END

Common Mistakes

❌ ลืม END
IF    ${x} > 0
    Log    positive
# ไม่มี END → Robot Framework error ทันที
✅ ปิดด้วย END เสมอ
IF    ${x} > 0
    Log    positive
END
source: RF User Guide — "The END token closes the IF/ELSE structure"
❌ ใช้ตัวเล็ก
if    ${x} > 0    # ← ผิด!
✅ UPPERCASE ทั้งหมด
IF    ${x} > 0
source: RF User Guide — keywords are case-sensitive
❌ Nested IF แต่ END ไม่พอ
IF    ${a} > 0
    IF    ${b} > 0
        Log    both positive
    END
# ลืม END ชั้นนอก → error
✅ END ทุก level
IF    ${a} > 0
    IF    ${b} > 0
        Log    both positive
    END
END    # ← END ของ IF ชั้นนอก
source: RF User Guide — each nested block requires its own END

🧠 ทดสอบความเข้าใจ — ตอบก่อนดูเฉลย

ข้อ 1. ถ้าเขียน IF ${score} >= 70 แล้วมี ELSE IF ${score} >= 90 ต่อมา — ถ้า score = 95 จะเข้า branch ไหน? เพราะอะไร?

ดูเฉลย

เข้า IF (>= 70) ทันที เพราะ RF ประเมินจากบนลงล่าง เจอ True ตัวแรกก็หยุด — ELSE IF >= 90 จะไม่ถูกประเมินเลย

แก้ไข: เรียง condition จากเข้มงวดที่สุดก่อน — ใส่ >= 90 ไว้บนสุด

ข้อ 2. Inline IF ต่างจาก block IF อย่างไร? ควรใช้อันไหนเมื่อไหร่?

ดูเฉลย

Inline IF เขียนบรรทัดเดียว ไม่มี END เหมาะกับ keyword เดียว / condition เดียว

Block IF ใช้เมื่อต้องการหลาย keyword ใน branch หรือมี ELSE IF

ข้อ 3. ทำไมต้องใส่ ' รอบ ${variable} เมื่อ compare กับ string?

ดูเฉลย

${env} ถูก resolve เป็น string ก่อน evaluate — ถ้าไม่ใส่ quote Python expression อาจ parse ผิด การใส่ quote ทำให้มั่นใจว่า compare string กับ string เสมอ
